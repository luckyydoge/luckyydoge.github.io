csci1680学习记录与放弃过程

发表于2024-03-26|更新于2024-05-16
|字数总计:2.1k|阅读时长:6分钟|阅读量:

课程网址：https://cs.brown.edu/courses/csci1680/f23/
我做的是fa23的版本

Project 0: Container envireonment setup

这个实验的目的是把实验环境配置好。

需要下载WSL2、Docker、X11 server，以及强烈建议下载vscode，方便写代码。

跟着实验文档一步步走下来就行，基本没什么坑。

文档里还提供了限制wsl使用内存的方法，方便物理机内存不够的学生顺利完成实验。

Project 1: Snowcast

实验目的：运用语言提供的api，实现基于tcp协议的服务器-客户机通信，客户机可通过连接到服务器接收服务器发送的歌曲信息流。歌曲的数据传输速度是128Kib/s(16KB/s)。客户机可通过定义好的命令与服务器进行通信，实现链接到不同的服务站、更改服务站等操作。服务器可输出当前所有工作站的信息，包括链接到各个工作站的客户机的ip地址和端口号，也可以输出到文件中保存。

该实验无基础代码，需要学生从零开始写。

The snowcast warmup

实验目的：搭建基础的客户机-服务器结构，熟悉网络编程，为后面的实现打基础。

IP地址和端口的区别

IP地址表示的是机器，端口表示的机器上运行的进程。

如果把一台机器比作是一个酒店，每个进程比作是酒店里的一间房子，IP地址就是酒店所在地，端口是每间房子的房间号。

Scoket

Scoket类似于一个文件描述符，用于本地机器上的进程与远方机器上的进程通信。类似于用pipe与其他进程通信。

实现步骤：

  1. 服务器选择一个接口作为监听接口，当客户机链接到监听接口时随机分配一个端口，通过随机分配的端口与客户机建立联系。（本实验中监听端口为16800）
  2. 客户机建立链接后向服务器发送特定结构，服务器接收后返回welcome，客户机接收welcome后将其中包含的数据打印出来。
  3. 更改程序结构使其符合测试要求，编写Makefile实现一键编译。

Server Design

要求服务器可以多线程地运行多个工作站，同时处理多个客户机的链接请求。当客户机链接超时时断开连接。

数据结构

需要存储client和station的信息，设计client_info和station_info结构存储。
client_info由：连接接口、IP地址、udp端口构成，使用map与其收听的station序号映射。
stattion_info由：收听的连接接口数组、同步锁、文件构成，使用map用序号映射此结构。

工作流程

  1. 进程启动时从命令行读取参数，使用goroutine创建station（执行1.1），创建对应的监听接口。
    1. 统计输入的文件数量，为每个文件用goroutine创建对应的station，将station序号与其映射，随后开始从文件读取数据并向链接数组写入。
  2. 用goroutine创建线程处理client的链接申请。
    1. 对于每个链接申请都创建一个goroutine去处理
    2. 首先接收Hello，发送Welcome，接收SetStation，发送Announce
    3. 将接受到的信息存储到client_info，将其与对应station序号映射，添加conn到station的链接数组
    4. 等待client发送其他命令并处理
  3. 主线程处理输入的命令。

Client Design

工作流程

  1. 启动时从命令行读取参数，将其通过Hello发送给server
  2. 接收到Welcome后，打印接收到的信息
  3. 准备从命令行接收命令，由此决定下一步行为

为什么放弃

首先绝对是低估了这个课程的难度和高估了自己的能力，以为自己能够在一个月内把这个课程的所有实验写完。事实上如果我想要写完这个课程的所有实验估计需要两个月的时间。

其次是自己学习节奏和学习方式的问题。之前的学习（包括s081、cs61b、csci0300）都是没有计划的在学习。s081好一点，每天都有要完成的任务，但是还是不清晰不明确，而且没有根据自己的能力制定任务。在学习其他课程时恨不得把每天的全部时间都投入到学习中，即使学到头昏脑胀也不休息，这样子不但效率不高，而且导致生活处在失控中：许多该做的事（例如跑步、看书）都没做。
这种失控感让我的学习效率更低，于是就花更多时间在学习上，生活的失控就越严重。这样的正反馈使得失控感不断增加，最终累计到一定程度失控感直接将我击溃了，我开始摆烂：每天除了刷视频就是看小说。

理所当然的，在ddl过后，我没有完成这门课程，甚至本应开始下一阶段的学习时，我还在摆烂。

反思

我崩溃的关键应该是课程太难，当时写IP的实验写了一个星期仍然没有头绪，导致直接开始摆烂了。也就是说，我应该选择垫垫脚就能达成的目标而不是要跳起来才能完成的目标。后者是要在状态好的情况才能完成，但我当时状态不好，ddl又临近，还有各种作业的压力使我选择了逃避。如果当时我能意识到这个目标太难，选择调整目标的话或许就能避免这次崩溃。

状态差的原因可能是生活的失控感和长时间学习的压力。那时候我几乎把所有时间都投入到了这个课程当中，运动游戏什么都没接触，导致不知不觉中就开始逃避学习，在学习时也无法保持高效率。

最近再看Maki的视频，里面提到了日拱一卒。日拱一卒的意思是每天都要学习，同时把时间线拉长，去关注一年中的总学习时间。如果从以年为单位的角度来看，某一天学多点少点其实并不能对总数产生很大的影响，但如果一段时间不学习就会造成总学习时间明显减少。

因此我需要保持一个良好的状态，持续的学习，不要再因为同样的事情崩溃。这意味着我需要学会与自我对话，关注自己的状态，在状态不好时学会调整。

效率也是一个很重要的点。50%的效率学8个小时和100%的效率学4个小时的效果是一样的，后者还节省了一半的时间去做其他事情，可以看出保持高效率能够在相同时间做更多事情。

现在想来之前学了8个小时的东西如果状态好的话其实可以节省2-3个小时，但是当时没有给自己设定每日任务，想的是学完了之后还要再继续向下学，导致学习状态有点像磨洋工，并不是以良好的状态去学习。

学习的效率是和精力挂钩的，当精力充足时学习效率自然很高，精力匮乏时即使看一个很简单的问题可能也看不懂问题在说什么。之前没有学会恢复精力，一直学是一直在消耗精力，自然学习效率不高。

期望与计划

希望在今年剩下的时间中不要再次崩溃，尝试实践日拱一卒，保持良好的学习节奏。

为此我需要培养良好的作息；学会恢复精力以及管理精力，让精力用在该用的地方；学会与自己对话，关注自己的状态，在状态不好时及时调整；学会制定任务，让自己明白这个时间段该学什么，以及在任务难度过高时及时调整任务。

就这样。
